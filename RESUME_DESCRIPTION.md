## 📌 版本三：详细版（适合高级开发者/技术面试官）

### 狼人杀多人实时对战游戏 | 全栈架构师
**技术栈**：Node.js v22、Express v4、Socket.IO v4、Vue 3、Pinia、MySQL 8、Tailwind CSS、Vite 5

**项目背景**：从零开始构建一款多人实时狼人杀游戏，解决实时通信、状态同步、断线重连等技术难点。

**技术架构**：

```
┌─────────────────────────────────────────────────────────────┐
│                        前端层 (Vue 3)                       │
│  LobbyView  │  RoomView  │  GameView  │  ChatBox  │  Stores │
├─────────────────────────────────────────────────────────────┤
│                      通信层 (Socket.IO)                      │
│              事件驱动 / 房间分组 / 自动重连                   │
├─────────────────────────────────────────────────────────────┤
│                        后端层 (Express)                     │
│  roomHandler  │  gameHandler  │  GameEngine  │  Models     │
├─────────────────────────────────────────────────────────────┤
│                      数据层 (MySQL + Cache)                  │
│                用户数据 / 游戏记录 / 房间缓存                 │
└─────────────────────────────────────────────────────────────┘
```

**核心技术亮点**：

1. **实时通信与状态同步**
   - 使用 Socket.IO 的房间分组功能（`socket.join()`），实现房间内广播
   - 设计事件驱动架构，将房间管理和游戏逻辑解耦为独立模块
   - 前端使用 Pinia 管理全局状态，后端事件触发自动更新

2. **断线重连机制**
   - **问题**：Socket.IO 每次连接生成新 `socket.id`，导致刷新后身份丢失
   - **方案**：引入 `userId` + `socketId` 双重标识，断线时保留记录
   - **实现**：断开连接时将 `socketId` 设为 `null`，30 秒超时后清理；重连时通过 `userId` 匹配恢复身份

3. **房主身份持久化**
   - **问题**：房主身份使用 `socket.id` 存储，刷新后无法识别
   - **方案**：新增 `hostUserId` 字段，使用 userId 标识房主
   - **实现**：重连时检查 `hostUserId`，自动恢复房主权限

4. **游戏引擎设计**
   - 角色配置：支持 4-12 人灵活配置，6 种角色（狼人、村民、预言家、女巫、猎人、守卫）
   - 夜晚行动：按顺序处理（守卫守护 → 狼人杀人 → 女巫用药），支持技能组合
   - 投票系统：实时统计票数，处理平票情况，触发猎人开枪技能

5. **自动复盘系统**
   - 数据采集：记录每次夜晚行动和投票（`gameHistory` 数组）
   - 报告生成：游戏结束时调用 `generateReplayMessage()` 生成复盘报告
   - 消息展示：通过聊天系统展示，保留换行格式

**代码质量与工程实践**：
- 模块化设计：将游戏引擎、房间管理、数据库操作分离为独立模块
- 错误处理：完善的错误捕获和日志记录
- 性能优化：内存缓存减少数据库查询，前端懒加载优化首屏性能
- 代码规范：统一的命名规范和代码风格

**项目成果**：
- 完整实现狼人杀游戏核心玩法和房间系统
- 支持多人实时对战，消息延迟 < 100ms
- 断线重连保持身份，用户体验流畅
- 自动复盘功能提升游戏体验和可玩性
- 代码结构清晰，文档完善，易于维护和扩展
